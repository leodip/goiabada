This guide helps you quickly run Goiabada on your local machine for evaluation purposes. It's the fastest way to explore the features and test the authentication flows before planning a production deployment.

<Aside type="caution">
This setup uses HTTP without encryption and is suitable only for local testing. Don't use in production!
</Aside>

<Aside type="tip">
Goiabada Docker images support both **x86_64 (amd64)** and **ARM64** architectures. They run natively on Intel/AMD machines, Apple Silicon Macs, Raspberry Pi 4/5, AWS Graviton, and other ARM64 devices.
</Aside>

## Using the setup wizard (recommended)

The fastest way to get started:

<Tabs>
  <TabItem label="Linux (x86_64)">
    ```bash
    curl -LO https://github.com/leodip/goiabada/releases/latest/download/goiabada-setup-linux-amd64
    chmod +x goiabada-setup-linux-amd64
    ./goiabada-setup-linux-amd64
    ```
  </TabItem>
  <TabItem label="Linux (ARM64)">
    ```bash
    curl -LO https://github.com/leodip/goiabada/releases/latest/download/goiabada-setup-linux-arm64
    chmod +x goiabada-setup-linux-arm64
    ./goiabada-setup-linux-arm64
    ```
  </TabItem>
  <TabItem label="macOS (Apple Silicon)">
    ```bash
    curl -LO https://github.com/leodip/goiabada/releases/latest/download/goiabada-setup-darwin-arm64
    chmod +x goiabada-setup-darwin-arm64
    ./goiabada-setup-darwin-arm64
    ```
  </TabItem>
  <TabItem label="macOS (Intel)">
    ```bash
    curl -LO https://github.com/leodip/goiabada/releases/latest/download/goiabada-setup-darwin-amd64
    chmod +x goiabada-setup-darwin-amd64
    ./goiabada-setup-darwin-amd64
    ```
  </TabItem>
  <TabItem label="Windows">
    ```powershell
    Invoke-WebRequest -Uri "https://github.com/leodip/goiabada/releases/latest/download/goiabada-setup-windows-amd64.exe" -OutFile "goiabada-setup.exe"
    .\goiabada-setup.exe
    ```
  </TabItem>
</Tabs>

Select **"1. Local testing"** when prompted, choose your database, and the wizard generates a ready-to-use `docker-compose.yml`.

```bash
docker compose up -d
```

Access the admin console at: http://localhost:9091

## Verify it's working

```bash
# Check containers are running
docker ps | grep goiabada

# Test auth server health (should return: healthy)
curl http://localhost:9090/health && echo

# Test admin console health (should return: healthy)
curl http://localhost:9091/health && echo
```
